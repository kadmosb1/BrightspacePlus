Hoe voeg ik een inhoudsopgave toe aan een pagina?

Op een willekeurige plaats in het document kun je een inhoudsopgave toevoegen: 
<div id="inhoudsopgave" visible="true"><!-- Toon hier een inhoudsopgave --></div>

Naast de id kun je twee andere attributen toevoegen:

  • visible="true": Als dit attribuut is toegevoegd met deze waarde, is de inhoudsopgave bij het tonen van de pagina meteen zichtbaar. Als dit attribuut een andere waarde heeft of niet is toegevoegd, wordt de inhoudsopgave initieel onzichtbaar gemaakt en kan de gebruiker deze via een toggle-link alsnog zichtbaar maken.
  • niveau="3": Het niveau staat voor het maximum niveau waarop headers worden getoond in de inhoudsopgave. Als het niveau is ingesteld op 3, dan worden h2 en h3 in de inhoudsopgave getoond (maar worden de met h4 gemarkeerde paragrafen dus niet getoond). Als waarde kun je alleen kiezen voor 2, 3 of 4.

Het toevoegen van de JavaScript

Als je een blended element (mediasite-filmpje, wooclap-vragenlijst, download-link etc.) toe wilt voegen aan de pagina, is het belangrijk dat aan de bovenkant van de pagina code wordt toegevoegd. Je moet daarvoor de pagina bewerken ('...' > 'Edit'). Als je de pagina bewerkt, kun je kiezen voor '</>':

Bovenaan elke pagina moet de volgende code worden toegevoegd (een paar elementen, zoals bijv. <html>, <head> en <body>, zijn bovenaan de pagina waarschijnlijk al toegevoegd):

<!DOCTYPE html>
<html><head>

  <script language="JavaScript">
 
    var strRootDir = document.URL.substr (0, document.URL.lastIndexOf ('/'));
    var strScriptID = 'inline.v4';
    var strTimestamp = new Date ().toUTCString ();
    var strJSFileName = strRootDir + '/resources/' + strScriptID + '.js?v=' + strTimestamp;
    var elmScript = document.createElement ('script');
    elmScript.setAttribute ('src', strJSFileName);
    elmScript.setAttribute ('async', false);
    elmScript.setAttribute ('type', 'text/javascript');
    elmScript.id = strScriptID;
    document.head.appendChild (elmScript);
   
    function logMessage (strType, strFileName, blnIsError) { 

        if (blnIsError) {
            console.log ('Error loading ' + strType + '-file \'' + strFileName + '\'');
        }
        else {
            console.log (strType + '-file \'' + strFileName + '\' loaded');
        }
    }

    elmScript.addEventListener ("load", () => { logMessage ('javascript', strJSFileName, false); });
    elmScript.addEventListener ("error", (ev) => { logMessage ('javascript', strJSFileName, true); });

  </script>

</head>

Opmaak van de pagina

Een hoofdstuk

Als je een grotere titel wilt plaatsen boven een tekst (die wordt genummerd met 'Hoofdstuk x'), kun je <h2> gebruiken:

Hoofdstuk x - Demo - Dit is een hoofdstuk

Een paragraaf

Als je een kleinere titel wilt plaatsen boven een tekst (die wordt genummerd met x.y), kun je <h3> gebruiken:

x.y - Dit is een paragraaf

Een subparagraaf

Als je een nog kleinere titel wilt plaatsen boven een tekst (die wordt genummerd met x.y.z), kun je <h4> gebruiken:

x.y.z - Dit is een subparagraaf

Je kunt drie attributen toevoegen aan een hoofdstuk of (sub-)paragraaf:

  • nummering="no": Dit hoofdstuk of deze (sub-)paragraaf telt dan niet mee in de nummering en er wordt geen paragraafnummer toegekend aan het hoofdstuk of de paragraaf. Een voorbeeld: <h3 nummering="no">.
  • inhoud="no": Dit hoofdstuk of deze (sub-)paragraaf wordt dan niet getoond in de inhoudsopgave. Een voorbeeld: <h4 inhoud="no">.
  • id: Om naar een hoofdstuk of (sub-)paragraaf te kunnen verwijzen, moet expliciet een id worden toegevoegd. Een voorbeeld: <h4 id="opdracht-garage">.
  • type: Dit type wordt gebruikt in een interne link en wordt toegevoegd aan de zichtbare titel. Een voorbeeld (zoals hierboven): <h2 type="demo">.

Hoe voeg ik een blokje toe, waarin evt. studietijd en navigatie worden getoond?

Bovenaan elk element van het type h2, h3 of h4 (bovenaan een hoofdstuk of (sub-)paragraaf) kan een blokje worden getoond met studietijd en/of navigatie. Wat is er nodig om zo'n blokje te tonen?

  • navigatie="yes": Als dit attribuut met deze waarde is toegevoegd, wordt rechts in dit blokje navigatie getoond die kan verwijzen naar een vorige paragraaf, naar de inhoudsopgave (als die bestaat) of naar een volgende paragraaf. De verwijzing naar een vorige of volgende paragraaf is alleen actief als er voorafgaand aan (voor vorige) of na (voor volgende) de huidige paragraaf één of meer paragrafen bestaan waarvoor ook 'navigatie="yes"' is ingesteld.
  • studietijd: Als studietijd wordt het aantal minuten opgegeven. Dit aantal minuten wordt omgerekend naar (halve) uren, kwartieren en minuten en getoond in dit blokje.
  • studietijd="som": Idem als bij studietijd, maar nu voor de optelling van de paragrafen die vallen onder dit hoofdstuk of deze paragraaf.
  • studietijd="totaal": Idem als bij studietijd, maar nu voor de hele pagina.
  • studietijd="extra: 20": Eventueel kan met de toevoeging 'extra' worden aangegeven dat de stof in de daarna volgende paragraaf facultatief is.

Stel dat je die verschillende combinaties als volgt opneemt op je pagina:

<h2 studietijd="totaal">Hoofdstuk met de optelling van alle studietijden op de hele pagina</h3>
<h3 studietijd="som">Paragraaf met de optelling van de onderliggende studietijden</h3>
<h4 studietijd="extra: 121" navigatie="yes">Subparagraaf met een studietijd van 121 minuten en navigatie</h4>
<h4 studietijd=75>Tweede subparagraaf met alleen een studietijd van 75 minuten</h4>
<h3 studietijd="som">Paragraaf zonder onderliggende studietijd</h3>
<h4 navigatie="yes">Subparagraaf met alleen navigatie</h4>

Dan wordt het volgende getoond:

Hoofdstuk met de optelling van alle studietijden op de hele pagina

Paragraaf met de optelling van de onderliggende studietijden

Subparagraaf met extra studietijd van 121 minuten en navigatie

Tweede subparagraaf met alleen toetsstof van 75 minuten

Paragraaf zonder onderliggende studietijd

Paragraaf met alleen navigatie

Hoe voeg je links toe aan een pagina?

Voor het toevoegen van een link kun je gewoon gebruik maken van de standaard functionaliteit van BrightSpace. Eventueel kun je de extra attributen toevoegen die hieronder worden beschreven.

Externe links

Voor een externe link kun je een start- en einddatum opgeven:

  • start="2024/01/14": Als je deze startdatum toevoegt, wordt de link pas actief vanaf die datum.
  • finish="2024/01/31": Als je deze einddatum toevoegt, wordt de link getoond tot en met die datum.
  • Zijn beide data ingevuld, dan is de link alleen actief vanaf de startdatum tot en met de einddatum.

Stel dat je onderstaande link hebt toegevoegd aan je pagina:

<a href="https://lelij.com" start="2024/01/14">Pagina van Karel J. van der Lelij</a>

Dan wordt deze link als volgt getoond en wordt deze link in een nieuwe tab geopend:

Pagina van Karel J. van der Lelij

Demo - Interne links

Voor een interne link hoef je geen attributen toe te voegen, maar het mag wel:

  • title: Deze title vervangt de zichtbare inhoud van de link. %t in deze title wordt vervangen door het type van de aangewezen link en %p wordt vervangen door het nummer van de paragraaf van de aangewezen link. In de eerste link wordt verwezen naar paragraaf x (de vorige paragraaf). Omdat het attribuut 'type' van deze paragraaf de waarde "demo" heeft, wordt %t in deze titel vervangen door het woord "demo". In de tweede link wordt verwezen naar paragraaf x (de huidige paragraaf). Omdat voor deze paragraaf geen type is gedefinieerd, wordt %t in de zichtbare titel vervangen door het woord "paragraaf".

Stel dat je onderstaande links hebt toegevoegd aan je pagina (omdat de eerste link verwijst naar een paragraaf zonder type, wordt %t vervangen door 'paragraaf':

<a href="#extern" title="%t %p">x</a>
<a href="#intern" title="%t %p">x</a>

Dan wordt deze link als volgt getoond en wordt deze link in een nieuwe tab geopend:

Het toevoegen van een Mediasite-, MS Stream- en/of YouTube-vid

De wrapper voor een video is het meest ingewikkelde onderdeel dat je kunt gebruiken. Elke video op een pagina kun je namelijk ook nog automatisch op laten nemen in een lijst met video's die op de pagina worden getoond. Om zo'n lijst met video's toe te voegen, neem je - op de plek waar je die lijst met video's wilt opnemen - de volgende code op:

<div class="video-list-wrapper"><!-- Toon links naar gebruikte video's op deze pagina --></div>

Om een video toe te voegen (die ook in de lijst wordt getoond), gebruik je de volgende code:

  • Voor Mediasite: 
    <div mediasite-id="f9e8ef2a38184d158b5b8d1f115106b31d" title="Een %t-filmpje in paragraaf %p"></div>
  • Voor MS Stream: 
    <div stream-id="73103235-5299-4a64-9927-c13da4e6ff66" account="kjvdlelij_hhs_nl" title="Een %t-filmpje in paragraaf %p"></div>
    <div stream-id="e38372e9-57cf-460c-ab29-0f423d3e014f" site="Filmopnames_groups" title="Een %t-filmpje in paragraaf %p"></div>
  • Voor Mediasite: 
    <div youtube-id="ZDpQQj3r_W8" title="Een %t-filmpje in paragraaf %p" visible="true"></div>

De gebruikte attributen hebben de volgende betekenis:

  • mediasite-id: De id die verwijst naar een vid in MediaSite (zoals in https://hhs.mediamission.nl/Mediasite/Play/f9e8ef2a38184d158b5b8d1f115106b31d).
  • stream-id: De id die verwijst naar een vid in MS Stream (zie paragrafen x en x in het eerste kader hieronder).
  • account (alleen bij MS Stream): Het account dat in de link wordt gebruikt om in OneDrive te kunnen verwijzen naar de juiste Stream (zoals in https://dehaagsehogeschool-my.sharepoint.com/personal/kjvdlelij_hhs_nl/_layouts/15/Embed.aspx?id=74bfde9e-f867-462f-b623-adb11337cfb2).
  • site (alleen bij MS Stream): De site die in de link wordt gebruikt om in SharePoint of Teams te kunnen verwijzen naar de juiste Stream (zoals in https://dehaagsehogeschool.sharepoint.com/sites/BP2526_groups/_layouts/15/Embed.aspx?id=bdb30083-2ab4-4de1-9d8a-7c20520372c6).
  • youtube-id: De id die verwijst naar een vid in YouTube (zoals in https://www.youtube.com/watch?v=ZDpQQj3r_W8).
  • paragraph: De paragraaf waarbinnen de video wordt getoond (Hieronder is een video toegevoegd aan paragrafen x, x en x). Dit attribuut is overbodig, omdat dit attribuut wordt overschreven door de automatische nummering van hoofdstukken en (sub-)paragrafen.
  • title: Deze bevat een korte beschrijving van de inhoud van de video. %t en %p in deze title verwijzen naar respectievelijk het type en het nummer van de paragraaf waarin dit element is opgenomen en worden vervangen door deze informatie.
  • visible="true": Als dit attribuut is toegevoegd met deze waarde, is de video (of de video-list) bij het tonen van de pagina meteen zichtbaar. Als dit attribuut een andere waarde heeft of niet is toegevoegd, wordt de video (of de video-list) initieel onzichtbaar gemaakt en kan de gebruiker deze via een toggle-link alsnog zichtbaar maken.

Het toevoegen van een Wooclap en/of een Wooflash (quiz of proeftoets)

Om een Wooclap- en/of Wooflash-vragenlijst toe te voegen, gebruik je de volgende code:

<div wooclap-dir="PR210/questionnaires/619626f40d21011226ce5d48" 
     wooflash-id="3ab639d3-ee25-48e9-9105-253aea5d948b" wooclap-werkvorm="het hoorcollege"></div>

<div wooflash-complete="3ab639d3-ee25-48e9-9105-253aea5d948b"></div>

<div wooclap-dir="PR210/questionnaires/619626f40d21011226ce5d48" type="wooclap-vraag"></div>

De gebruikte attributen hebben de volgende betekenis:

  • wooflash-id: De id van een vragenlijst in Wooflash (https://app.wooflash.com/study/3ab639d3-ee25-48e9-9105-253aea5d948b).

  • visible="true": Als dit attribuut is toegevoegd met deze waarde, is de quiz bij het tonen van de pagina meteen zichtbaar. Als dit attribuut een andere waarde heeft of niet is toegevoegd, wordt de quiz initieel onzichtbaar gemaakt en kan de gebruiker deze via een toggle-link alsnog zichtbaar maken.

Het toevoegen van de uitwerking van een opdracht

Als je programmacode als uitwerking van een opdracht toe wilt voegen, gebruik je de volgende code:

<pre>public class Main {

    public static void main(String[] args) {
        System.out.println ("Hello world!");
    }
}</pre>

Als je 'gewone' tekst toe wilt voegen, gebruik je een div:

<div class="blended-uitwerking">
    Dit is een test met een lijst
    <ul>
        <li>1</li>
        <li>2</li>
    </ul>
</div>

In de zichtbare titel boven de uitwerking worden gegevens verwerkt van het hoofdstuk of de (sub-)paragraaf waarin de uitwerking voorkomt. De structuur van deze titel is "Klik hier om de uitwerking van %t %p (on-)zichtbaar te maken". In deze titel wordt %t vervangen door de waarde van het attribuut 'type' van de paragraaf en %p door de waarde van het attribuut 'paragraph' van het hoofdstuk/de (sub-)paragraaf waarin de uitwerking voorkomt. Het attribuut paragraph wordt automatisch toegekend. Het attribuut 'type' moet je zelf toevoegen aan dit hoofdstuk/deze (sub-)paragraaf. Als het attribuut 'type' niet is gedefinieerd, wordt %t op basis van het type van het hoofdstuk/de (sub-)paragraaf vervangen door 'hoofdstuk' of 'paragraaf'.

Het gebruikte attribuut heeft de volgende betekenis:

  • paragraph: Dit attribuut is overbodig geworden, omdat dit attribuut wordt overschreven door de automatische nummering van hoofdstukken en (sub-)paragrafen. De paragraaf waarin de opdracht (en dus ook de uitwerking daarvan) wordt beschreven (op deze pagina is de uitwerking van de opdracht toegevoegd aan paragraaf x). Deze paragraaf wordt gebruikt in de titel van de uitwerking.
  • visible="true": Als dit attribuut is toegevoegd met deze waarde, is de uitwerking bij het tonen van de pagina meteen zichtbaar. Als dit attribuut een andere waarde heeft of niet is toegevoegd, wordt de uitwerking initieel onzichtbaar gemaakt en kan de gebruiker deze via een toggle-link alsnog zichtbaar maken.
  • title="Een titel": Als deze titel is ingevuld, dan wordt die in de toggle-balk getoond.

Als je bovenstaande code-voorbeelden (met toegevoegde paragraaf) gebruikt voor de uitwerking van een opdracht worden de wrappers met de uitwerking als volgt getoond:

Het toevoegen van een PDF-viewer

Je kunt een PDF-document ook direct in je pagina zichtbaar maken. Daarvoor gebruik je de volgende code (die hier wordt gegeven voor zowel een absolute als voor een relatieve link):


<div pdf-root="relative" pdf="media/onderhoud/Onderhoud - Voorbeeld.pdf" page="4" visible="true"></div>

De gebruikte attributen hebben de volgende betekenis:

  • pdf: De link naar de PDF-file die moet worden getoond.
  • pdf-root: Is de link absoluut (startend met https://..., https://... of file://...) of relatief (en wordt er dus voor de link geplakt).
  • page: Nummer van de pagina in het PDF-document die moet worden geopend als het document wordt ingeladen. Als je gebruik wilt maken van dit attribuut, moet dit worden gecombineerd met het attribuut en de waarde visible="true" (omdat anders de PDF niet direct wordt geopend en daardoor het document niet op de geselecteerde pagina wordt getoond).
  • visible="true": Als dit attribuut is toegevoegd met deze waarde, is het PDF-document bij het tonen van de pagina meteen zichtbaar. Als dit attribuut een andere waarde heeft of niet is toegevoegd, wordt het PDF-document initieel onzichtbaar gemaakt en kan de gebruiker deze via een toggle-link alsnog zichtbaar maken.

Het toevoegen van een FAQ

Om een FAQ op je pagina toe te voegen, gebruik je de volgende code:

<div header="4" werkvorm="het hoorcollege" title="Een voorbeeld van een FAQ">
    <div>
        <div>Vraag 1</div>
        <div>Antwoord 1</div>
    </div>
    <div>
        <div>Vraag 2</div>
        <div>Antwoord 2</div>
    </div>
</div>

Deze FAQ wordt getoond in een vaste opmaak. De gebruikte attributen hebben de volgende betekenis:

  • werkvorm: Deze werkvorm wordt verwerkt in de toelichting op de FAQ.
  • header: Het nummer voor de paragraaf (die default op 3 voor H3 staat).
  • title: De titel die boven de FAQ wordt getoond (standaard is dat FAQ).

<div site="AgileOOProgrammingSemesterSE-S2_groups-Docenten" pptx-id="34c09546-2159-4bd6-82a5-f73bd68cd84f"></div>

  • pptx-id: Het id dat in de link wordt gebruikt om in OneDrive, SharePoint of Teams te kunnen verwijzen naar de juiste file (zoals in https://dehaagsehogeschool.sharepoint.com/sites/SE2-2021-2022_groups/_layouts/15/Doc.aspx?sourcedoc={703c7cf7-3ce7-4f57-bee9-b2def3aa70c3}&action=embedview&wdAr=1.7777777777777777).

Als dit attribuut een andere waarde heeft of niet is toegevoegd, wordt de presentatie initieel onzichtbaar gemaakt en kan de gebruiker deze via een toggle-link alsnog zichtbaar maken.
